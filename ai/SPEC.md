# 프로젝트 명세 (SPEC)

## 1. 개요
'Chroma Hacker'는 사용자 카메라 피드를 실시간으로 분석하여 특정 채도 이상의 영역을 식별하고, 해당 영역에만 실시간으로 ASCII 문자를 오버레이하는 단일 페이지 웹 애플리케이션이다.

## 2. 주요 기능
- **카메라 스트리밍**: `getUserMedia` 기반 실시간 비디오 렌더링
- **채도 마스킹**: 실시간 픽셀 분석을 통해 RGB를 HSL로 변환, 사용자 설정 기준 임곗값(Threshold) 이상 픽셀 추출
- **ASCII 오버레이**:
  - 추출된 고채도 영역에만 텍스트 렌더링
  - 픽셀 밝기(Lightness)에 따른 ASCII 문자 매핑 (예: `@#S%?*+;:,..`)
  - 주변부(저채도 영역)은 원본 카메라 영상을 그대로 렌더링
- **사용자 제어 패널**:
  - 채도 임곗값 (Saturation Threshold) 슬라이더 (0-100%)
  - ASCII 밀도/해상도 조절 슬라이더
  - ASCII 반영 딜레이 슬라이더 (실시간 ~ 60프레임 지연)
  - 글자 색상 선택 옵션 (원본 색상 투영 또는 지정된 단색)
  - 배경 화면 숨기기 체크박스 (카메라 원본 영상을 숨기고 검은 배경에 텍스트만 렌더링)
  - 텍스트 배경 가림막 없애기 체크박스 (텍스트 뒤의 검은색 박스를 없애 원본 영상 위에 텍스트만 오버레이, 기본값 활성)
  - 채도 비례 발광(Bloom) 효과 체크박스 (채도 0~100% 구간에 비례하여 텍스트 그림자가 해당 색상으로 빛나는 발광 효과 적용)

## 3. 기술 사양 및 스택
- HTML5, CSS3 (바닐라 CSS), Vanilla JavaScript (ES6+)
- Canvas API
- **성능 최적화**: `requestAnimationFrame` 사용 및 불필요한 가비지 컬렉션(Object creation) 최소화
- **UI 디자인**: 카메라 화면을 가리지 않는 하단 또는 우측에 위치한 반투명(Glassmorphism) 컨트롤 패널 (다크 모드 적용)

## 4. 구조화 방식
- `index.html`: UI 컨트롤 요소 및 캔버스 렌더링 부모 레이아웃.
- `style.css`: 반투명 UI 및 전반적인 레이아웃 정의.
- `js/constants.js`: 고정형 매직 넘버(ASCII 문자열 목록, 캔버스 초기 크기 등) 관리 파일.
- `js/script.js`: 웹캠 제어, 캔버스 드로잉 루프, HSL 변환 수학 로직, 이벤트 리스너 통합.

## 5. 코딩 컨벤션
- 변수, 함수: `camelCase`
- 상수: `UPPER_SNAKE_CASE`
- 식별자는 영어로 하되, 주석 및 설명은 반드시 한국어로 작성(`ai/GEMINI.md` 준수).
- `null`/`undefined` 방어 로직 설계 (Optional Chaining 적용).
